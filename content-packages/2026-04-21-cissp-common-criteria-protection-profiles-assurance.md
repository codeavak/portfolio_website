## 2. Research Summary

**Common Criteria:**

- International standard for evaluating security properties of IT products and systems.
- Developed as a broader successor to older national evaluation models such as TCSEC.
- Focuses on both functional requirements and assurance requirements.

**Protection Profile (PP):**

- Implementation-independent set of security requirements for a class of products.
- Useful in procurement and standardizing expectations before selecting a vendor.

**Security Target (ST):**

- Product-specific statement defining what the evaluated product claims to provide.
- Includes scope, assumptions, threats, security functions, and assurance claims.

**Evaluation Assurance Levels (EAL1-EAL7):**

- Describe the rigor and depth of evaluation assurance.
- They are not simple security quality rankings.
- Higher EAL means more assurance rigor, not universal superiority in every deployment.

**CISSP exam angle:**

- Protection Profile = generalized requirement baseline.
- Security Target = specific evaluated product claim set.
- Common Criteria = evaluation framework.
- EAL = assurance rigor.
- Exam often tests the limit of certification-based trust.

## 4. LinkedIn Post

---

Security certifications are useful.

They are also easy to misunderstand.

That is why Common Criteria matters.

It gives a structured way to evaluate security products, but it does **not** mean a certified product is automatically secure in your environment.

The key ideas:

**Protection Profile**: the generic requirement baseline for a class of products.

**Security Target**: the product-specific claim describing what one evaluated product actually provides.

**Evaluation Assurance Level (EAL)**: how rigorous the evaluation evidence was. Not a universal security score.

This is the mental model I use:

- Common Criteria improves confidence
- It does not remove deployment responsibility
- It does not guarantee correct configuration
- It does not eliminate operational risk

That is as true in real procurement as it is on the CISSP exam.

If a team treats certification as immunity, they are already making a security mistake.
