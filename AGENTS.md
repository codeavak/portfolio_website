# Blog content operating rules

## Goal
Use this repository to create high-quality technical content for my Jekyll blog and related social content.

## My background
- Senior software engineer
- Security-minded developer
- Experience with C#, ASP.NET, SQL Server, APIs, web applications, cloud, and secure development practices
- Interested in career growth, software engineering excellence, discipline, self-improvement, and long-term credibility
- I want to attract recruiters, hiring managers, engineers, cybersecurity professionals, and ambitious professionals

## My tone
- Intelligent
- Calm
- Credible
- Reflective
- Practical
- Slightly polished but natural
- Confident, not arrogant
- Professional, not corporate and lifeless

## My content pillars
- Software engineering lessons
- Security-minded development
- Career growth
- Leadership without authority
- Building reliable systems
- Technical discipline
- Professional resilience
- Certifications and continuous learning
- AI in development used responsibly
- Lessons from real work, not empty theory

## Research rules
- Use current, credible information when the topic benefits from freshness.
- Distinguish clearly between established principles and recent developments.
- Do not invent statistics, citations, case studies, or personal stories.
- Avoid fluff, fake thought leadership, and generic motivational filler.
- Prioritize long-term credibility over short-term virality.

## Jekyll rules
- Create blog posts in `_posts/`.
- Use filenames in `YYYY-MM-DD-slug.md` format.
- Use front matter with:
  - layout: post
  - title
  - date
  - categories
  - tags
  - excerpt
  - image
- Save generated images under `assets/generated/YYYY/MM/slug/`.
- Insert one hero image near the top and up to two inline images where they improve readability.
- Do not commit changes unless I explicitly ask.

## Deliverables for each run
Create these files:

1. Jekyll post:
   - `_posts/YYYY-MM-DD-slug.md`

2. Content package:
   - `content-packages/YYYY-MM-DD-slug.md`

The content package must contain this exact order:
1. Positioning summary
2. Research summary
3. Detailed blog post
4. LinkedIn post
5. Extra content assets

## Blog standards
- Aim for 1200–1800 words unless I say otherwise
- Strong intro hook
- Clear H2/H3 structure
- Short paragraphs
- Practical examples when useful
- Original and thoughtful, not templated
- Subtle CTA
- End with meta description and 5 SEO keyword ideas

## LinkedIn standards
- Shorter than the blog
- Strong opening hook
- Easy to skim
- Natural closing discussion prompt
- No cringe
- Max 5 hashtags

## Execution preference
- First choose the strongest theme if I did not provide one.
- Then research.
- Then draft the full package.
- Then create or prepare images.
- Then wire everything into the Jekyll post.
- Finally show me:
  - the chosen topic
  - files created
  - a short summary
  - the diff
